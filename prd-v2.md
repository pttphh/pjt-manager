# 프로젝트 관리 툴 — PRD v2 (2026.07 재설계)

> v1(회의 중심 구조) 폐기 후 전면 재설계. 본 문서가 유일한 기준이며, 기존 PRD·스키마·프롬프트 문서는 참조하지 않는다.

## 개요
경영자와 소수 팀원이 사용하는 내부 업무 관리 웹앱.
PJT(프로젝트) → Tasks(작업) → Todo 3계층으로 업무를 관리하고,
Task 단위로 배포 → Todo 단위로 진행사항을 체크하는 도구.
**PC 전용** (모바일 반응형 미지원). 로그인/계정 시스템 없음.

## 기술 스택
- Frontend: Vite + React 18 + TypeScript + Tailwind CSS v3
- DB: Supabase (anon key, RLS 비활성화)
- 라우팅: React Router v6
- 배포: Vercel / 버전관리: GitHub

## 인증
- 단일 비밀번호 입력 → 환경변수 `VITE_APP_PASSWORD` 비교
- 일치 시 sessionStorage `authenticated='true'` 저장 후 메인 진입
- 역할 없음. 전원 동일 권한. 역할 선택 화면 없음.

인라인 관리 팝업(목록 + 각 항목 수정/삭제 + 하단 추가 입력)으로 처리.

## 업무 트리와 상태 체계

```
PJT (구분 1개 필수, 태그 0~n개)
 └── Task (소속 PJT 고정, 날짜·멤버·결정&전달사항 보유)
      └── Todo (담당자 복수, PJT 개별 매칭 가능)
```

### Todo의 PJT 개별 매칭 (핵심 규칙)
- Todo 생성 시 기본 PJT = 소속 Task의 PJT
- 드롭박스로 **동일 구분 내 '미진행' 또는 '진행중' 상태의 다른 PJT**로 변경 가능
- 따라서 Todo는 `task_id`(작성 출처)와 `project_id`(매칭 PJT)를 모두 가진다
- Tasks 배포 탭: Todo의 PJT와 무관하게 **Task 단위**로 묶어 표시
- PJT 세부화면: Task와 무관하게 **project_id 기준**으로 Todo 수집

### 상태 정의

| 엔티티 | 상태 | 전환 |
|---|---|---|
| PJT | 미진행(노랑) / 진행중(파랑) / 보류(빨강) / 완료 | PJT 세부화면 상단 드롭박스. 완료 시 PJT 관리 목록에서 제거, 설정창에서 열람 |
| Task | 미배포 / 배포됨 | Task 팝업 하단 버튼으로 양방향 전환. 배포 시 deployed_at 기록 |
| Todo | 미진행 / 체크 / 완료 | 아래 규칙 참조 |

### Todo 상태 전환 규칙
- 미진행 → 체크: Todo 체크 화면에서 진행사항 메모 저장 시
- 체크 → 완료: Todo 체크 화면 '완료로 변경' 버튼
- **PJT 세부화면 체크박스**: 상태 무관 자유롭게 체크/해제 가능 (메모 불필요, 미배포 Task의 Todo 포함)
  - 체크 시 → 완료
  - 해제 시 → 메모 이력이 있으면 '체크', 없으면 '미진행'으로 복귀
- 체크 → 미진행: Todo 체크 화면에서 되돌리기 가능
- 진행사항 메모는 **누적 저장**, 화면에는 최신 메모만 표시

### 화면 노출 규칙
- Todo 체크 화면: **배포 여부 무관 전체** Task의 Todo 흐름 (각 Todo 옆 배포/미배포 뱃지, 완료 Todo 제거, 전 Todo 완료 시 그룹도 제거)
- PJT 세부화면 Todo 목록: 미배포 Task의 Todo **포함** 전체 표시
- 배포됨 목록(Tasks 배포 탭): 진행 중 Task만 (전 Todo 완료 시 제거, PJT 세부화면에서 열람)

### "기타" 상설 Task
- PJT 등록 시 "기타" Task 자동 생성 + 자동 배포됨 상태
- 단발성 Todo를 Task 작성 없이 바로 붙이는 용도

### 배포의 의미
- 계정·알림 없음 → 배포는 상태 전환일 뿐 (Todo 체크 화면엔 배포 무관 전체 노출, 배포/미배포는 뱃지로 구분)
- 실제 전파는 운영 관행(단톡방 공지 등)으로 처리. 알림 연동은 추후 과제.

## 화면 구성

### 공통 레이아웃
좌측 사이드바 + 우측 본화면. 사이드바 너비 드래그 조절(min 160 / max 320px, localStorage 저장).

**사이드바**: 앱 이름 / 전체(메인 3탭 복귀) / **구분별 트리** (구분 접기·펼치기 → 하위 PJT 목록).
PJT 클릭 시 → PJT 세부화면으로 이동.

### 1. 비밀번호 입력 (`/`)
중앙 카드: 앱 이름, 비밀번호 input, 입장 버튼, 오류 메시지.

### 2. 메인 (`/main`) — 3탭

#### 탭 A. Todo 체크
- 상단에 **구분 필터 칩** (전체 / Biz 일반 / 유맥 / …) — Todo의 project_id가 속한 구분 기준
- 상단 구간 "미진행 Todo": Task 단위 아코디언 (행 표기: `Task명 (배포 M/D) — 프로젝트명`, 우측 미진행 건수)
  - 펼치면 미진행 Todo + 메모 입력창 + 저장 (저장 시 체크로 전환, 하단 이동)
- 하단 구간 "체크됨": 동일하게 Task 단위 묶음
  - Todo마다 최신 진행사항 메모 표시(날짜만, 시각 없음) + '완료로 변경' 버튼
  - 한 Task가 미진행·체크 Todo를 모두 가지면 양쪽 구간에 동시 노출 (각 구간엔 해당 상태 Todo만)

#### 탭 B. Tasks 배포
- 상단 "미배포(작성중)": 행 = `Task명 (작성 M/D) — 프로젝트명` + '내용보기' 버튼만
- 하단 "배포됨": 행 = `Task명 (배포 M/D) — 프로젝트명` + Todo n/m 완료 + '내용보기'
- 새 Task 작성 버튼 없음 (Task 작성은 PJT 세부화면에서)

#### 탭 C. PJT 관리
- 우측 상단 'PJT 등록' 버튼
- **태그별 컬럼**에 PJT명 카드만 나열 (노션 갤러리 스타일)
  - 카드 배경색 = PJT 상태: 미진행(노랑) / 진행중(파랑) / 보류(빨강)
  - 복수 태그 PJT는 각 태그 컬럼에 중복 노출
  - 맨 끝 **"태그 없음" 컬럼** (태그 미선택 PJT)
  - 완료 PJT는 미표시 (설정창에서 열람)
- 카드 클릭 → PJT 세부화면

### 3. PJT 세부화면 (`/project/:id`)
- 상단 정보 카드:
  - `[상태 드롭박스] PJT명` + 우측 편집 버튼 (편집 팝업 = PJT 등록 팝업과 동일 폼, **삭제 버튼 포함**)
  - 4줄 정보: ① 링크(온라인 주소, 클릭 시 새 창) ② 구분·태그 뱃지 ③ 기간(시작일 — 완료 예정일) ④ 멤버 칩
- 하단 좌: **Tasks** — '신규 Task 등록' 버튼 + Task 버튼 목록 (표기: `YY.MM.DD Task명`) → 클릭 시 Task 팝업
- 하단 우: **Todo** — 이 PJT에 매칭된(project_id 기준) 전체 Todo
  - 체크박스 리스트: 미완료 위 / 완료 아래(체크 + 취소선)
  - 체크박스 직접 체크/해제 가능 (상태 전환 규칙 참조)

### 4. Task 팝업 (보기/편집 겸용 단일 창)
신규 등록·기존 Task 클릭·Tasks 배포 탭 '내용보기' 모두 이 창. 기존 Task는 내용이 채워진 채 즉시 수정 가능.
- 헤더: "Task 작성" + 소속 PJT(고정) 표기
- Task명 / 날짜(기본=작성일, 지정 가능)
- 멤버: 기본 = PJT 멤버 전원, 태그 입력으로 추가·삭제
- 결정 & 전달 사항: 자유 텍스트
- Todo 행 (내용 넓게 / 담당자 / PJT / 삭제):
  - 담당자: 드롭박스 → **Task 멤버 중 체크박스 복수 선택**
  - PJT: 드롭박스 (기본=Task의 PJT, 동일 구분의 미진행·진행중 PJT로 변경 가능)
  - Todo 상태는 이 팝업에 표기하지 않으며 변경 불가
- 하단: **저장** + 상태에 따라 **'배포 완료'**(미배포일 때) 또는 **'미배포로 되돌리기'**(배포됨일 때) + **삭제** 버튼
- 삭제 시 하위 Todo·메모 연쇄 삭제, 확인창 필수 (PJT 삭제도 동일: 하위 Task·Todo 연쇄, 확인창)

### 5. PJT 등록/편집 팝업
- PJT명 / 기초 사항(자유 텍스트) / 구분(드롭다운, 필수) + ⚙ / 태그(칩 복수 선택) + ⚙ / 멤버(태그 입력) / 시작일 / 완료 예정일
- 상태 기본값 = 미진행. 저장 시 "기타" Task 자동 생성(배포됨)
- 편집 모드에는 삭제 버튼 포함

### 6. 설정 (경량)
- 완료된 PJT 목록 (열람 + 재활성화 + 삭제)
- 담당자(people) 목록 관리
